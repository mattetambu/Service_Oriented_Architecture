Parametri

Restituisce
Risultato della verifica, true in caso di successo e false altrimenti. Al Server sono necessari tre parametri per la sua esecuzione e se viene invocato con un numero inferiore di argomenti questa funzione richiede l'inserimento di quelli mancanti. Il controllo sui parametri necessari al Server viene effettuato nel seguente modo:
  • il primo deve essere un intero nell'intervallo [1024-65535] e rappresenta la porta di ascolto dell' Image_manipulation_server
  • il secondo deve essere un indirizzo IP e rappresenta l'indirizzo IP del Service_register_server
  • il terzo deve essere un intero nell'intervallo [1024-65535] e rappresenta la porta di ascolto del Service_register_server

void * control_thread_body ( void *  )

Corpo del thread di controllo dell' Image_manipulation_server.

Corpo del thread di controllo del Service_register_server.

Corpo del thread di controllo dell' Image_storage_server.

Il thread di controllo, fino alla ricezione del comando di terminazione, esegue ciclicamente le seguenti operazioni:

  • si pone in attesa di essere attivato (manualmente) per eseguire un comando
  • richiede l'inserimento del comando e verifica se è in grado di eseguirlo (altrimenti lo rifiuta)
  • se il comando viene accettato esegue l'operazione corrispondente

Corpo di tutti i threads di servizio del Service_register_server.

Corpo di tutti i threads di servizio dell' Image_storage_server.

Parametri
[out]thread_IDIntero che identifica il thread a cui è associata l'istanza della funzione. Ogni thread di servizio, fino alla ricezione del comando di terminazione, esegue ciclicamente le seguenti operazioni:
  • segnala la sua disponibilità a ricevere connessioni e si pone in attesa di essere attivato per servire un Client
  • riceve il nome del servizio richiesto dal Client e se questo non corrisponde ad un servizio fornito rifiuta al richiesta
  • se la richiesta del Client viene accettata riceve i parametri necessari per l'esecuzione del servizio
  • richiama il servizio richiesto (rotazione o riflessione dell'immagine) ottenendo il risultato della manipolazione
  • invia al Client il risultato dell'esecuzione del servizio
